## Admin guard

### `proxy.ts` — role kontrolü
Mevcut login proxy'sine `/admin/*` için ek `profiles.role = 'admin'` kontrolü eklendi. Admin değilse `/`'a redirect.

### `lib/auth/admin.ts` (yeni)
- `getAdminUser()` — `React.cache` ile request başına 1 kez profile.role çeker.
- `requireAdmin()` — server component/action'larda kullanılır, admin değilse `/giris?next=/admin`'e redirect.

**Karar:** Yazımlar service role client (`createAdminClient`) ile yapılıyor — sepetteki `reserved_stock` mantığıyla aynı. Admin RLS policy yazılmadı; `requireAdmin()` zaten kontrolü sağlıyor.

## Ürün yönetimi (asıl odak)

### `app/(admin)/admin/urunler/page.tsx`
- Server component, `createAdminClient()` (RLS bypass → pasif ürünler de görünür).
- Filtreler: `q` (product_name/fancy_name ilike), `label` (kategori dropdown), `filter` (no-price/no-stock/inactive).
- Sayfalama: 50/sayfa, `?page=N`.
- Tablo görünümü: görsel + kod/isim/tip + kategori + fiyat (inline) + stok (inline) + aktif (toggle) + Düzenle linki.
- Boş state: "Eşleşen ürün yok."

### `app/(admin)/admin/urunler/ProductRow.tsx` (client)
- Inline edit: `price` ve `stock` input'ları local state + `useTransition`.
- Blur'da `commitPrice`/`commitStock` → değer initial'dan farklı ve geçerliyse `updateProduct` çağrısı.
- Enter = blur, Escape = revert.
- Aktif toggle: butonu tıklayınca optimistic local state + server action.
- Visual feedback: saving (sarı border), saved (yeşil flash 1.5s), error (kırmızı border + flash).

### `app/(admin)/admin/urunler/[id]/page.tsx` + `ProductEditForm.tsx`
- Detay sayfası: tüm alanlar (price, compare_price, stock, label, image_url, description TR/EN, meta TR/EN, active/featured).
- `reserved_stock` readonly görüntüleme (sepetteki rezerve stok için).
- Form: `Section` (fieldset) gruplaması — Ticari / Genel / Açıklamalar / SEO.
- Submit → `updateProduct` → ✓ Kaydedildi / ✕ Hata feedback.

### `lib/admin/product-actions.ts`
- `"use server"` directive.
- `updateProduct(id, patch)` — `getAdminUser()` guard, input validation (price/stock/compare_price >= 0), service role update, cache invalidation.
- `toggleProductActive(id, active)` — `updateProduct` proxy.
- `bulkUpdateProducts(ids, patch)` — toplu price/stock/active update (UI'da henüz kullanılmadı, ileride toplu işlem için).
- `invalidateProductCaches(productName?)`:
  - `revalidateTag("products", "max")` — **Next 16'nın yeni 2-argümanlı API'sı** (`profile="max"` ile stale-while-revalidate).
  - `revalidatePath("/urunler" | "/" | "/urun/[name]")`.
  - `revalidatePath("/admin/urunler" | "/admin/urunler/[id]")`.
